### Actividad 1
Teniendo en cuenta el material de clase, responda las siguientes preguntas:

1. ¿Por qué el desarrollo de software no puede realizarse simplemente comenzando a programar?

   R:Porque primero hay que analizar las necesidades del cliente, planificar el proyecto, diseñar la solución y definir los requerimientos. Programar sin planificación puede generar errores y un sistema que no cumpla con lo que el usuario necesita.

2. ¿Qué significa que un desarrollo sea dirigido por un plan?

   R:Significa que antes de desarrollar el software se realiza un plan detallado con las etapas, tiempos, recursos y requerimientos que guiarán todo el proyecto.

3. ¿Cuáles son las ventajas de utilizar un plan de desarrollo?

   R:
  •	Permite estimar tiempos y costos.
	•	Organiza el trabajo en etapas.
	•	Hace el desarrollo más predecible.
	•	Facilita el control del proyecto.

4. ¿Qué críticas se hacen a los modelos tradicionales de desarrollo?

   R:
  •	Son muy rígidos.
	•	No permiten cambios fácilmente.
	•	Los errores de etapas iniciales se arrastran.
	•	El cliente ve el sistema recién al final.

5. ¿Por qué en la práctica muchas organizaciones combinan metodologías ágiles y modelos dirigidos por un plan?

   R:Porque así pueden mantener la organización del plan y al mismo tiempo adaptarse a cambios mediante prácticas ágiles.

### Actividad 5 – Caso práctico

**Características del proyecto:**
- Sistema relativamente pequeño
- Pocos usuarios
- Requerimientos claros
- Tiempo limitado

**Preguntas**

- ¿Qué modelo de desarrollo recomendaría? 

  R: El modelo en cascada (lineal secuencial).
- Justifique su respuesta.

  R:Porque el sistema es pequeño, tiene pocos usuarios y los requerimientos están claros, por lo que un modelo simple y secuencial resulta suficiente y más rápido.
- ¿Qué etapas principales tendría el desarrollo?

  R:
  1.	Análisis de requerimientos
	2.	Diseño
